# 沛素 per-sulii 官網｜免費部署方案規劃

> 目標：用 **Astro + Sveltia CMS + Cloudflare Pages**，把官網以**新台幣 0 元**（僅需既有網域年費）部署上線，並具備合作洽詢表單、流量分析、繁中／英文雙語，且**幾乎零維護**。

---

## 一、技術架構總覽

```
 內容編輯                建置與託管                  使用者
┌──────────────┐      ┌─────────────────────┐     ┌──────────┐
│ Sveltia CMS  │ ──→  │ GitHub（原始碼倉庫） │ ──→ │ 你的網域  │
│ 圖形後台發文  │ 推送 │        ↓ 自動觸發     │ CDN │ persulii │
└──────────────┘      │ Cloudflare Pages 建置 │ ──→ │  訪客     │
                      │ Astro 編譯成靜態 HTML │     └──────────┘
                      └─────────────────────┘
```

運作邏輯：你在 CMS 後台發文 → 內容自動存進 GitHub → Cloudflare Pages 偵測到變更、自動重新編譯 → 數秒內全球上線。整個過程不需要你碰程式碼，也沒有伺服器或資料庫要維護。

---

## 二、為什麼是這個組合

- **Astro 現在屬於 Cloudflare**：2026 年 1 月 Cloudflare 收購了 Astro 團隊，兩者整合度最高、長期支援無虞，框架維持開源免費。
- **靜態網站＝低維護、高安全、超快**：沒有資料庫與外掛，不會像 WordPress 需要定期更新修補、也不怕被駭。
- **Cloudflare Pages 頻寬不計費**：流量再大也不會被中斷或收費，適合品牌官網的流量起伏。
- **Sveltia CMS 給你 WordPress 般的後台**：它是 Decap CMS（前 Netlify CMS）的現代繼任者，設定檔相容，但更輕快、且能直接搭配 Cloudflare 與 GitHub 登入（Decap 舊版依賴的 Netlify Identity 已逐步淘汰，故改用 Sveltia 更穩妥）。

## 四、三項功能怎麼免費做到

### 1. 合作洽詢表單
- 採用 **Web3Forms**：在表單貼上一組免費 access key，訪客送出後直接寄到你指定信箱（如 `service@persulii.com` 或你的 Gmail）。
- 免後端、免資料庫；附蜜罐欄位防垃圾訊息。
- 若日後洽詢量變大或想要後台管理表單，再升級或改接其他服務即可。

### 2. 流量分析
- 主用 **Cloudflare Web Analytics**：與 Pages 同一個後台，一鍵開啟，看瀏覽量、來源、熱門頁面、裝置別，且隱私友善（歐盟 GDPR 無痛）。
- 若你想做更深入的行銷追蹤（轉換、UTM、事件），可**另外再掛 Google Analytics 4（免費）**並行，兩者不衝突。

### 3. 繁中／英文雙語
- 使用 **Astro 內建 i18n**：以網址前綴分流（`/`＝繁中、`/en/`＝英文），自動產生 hreflang，對 SEO 友善。
- Header 放語言切換鈕；CMS 後台可分別維護兩種語言內容。

## 五、部署流程（一次性設定，約半天）

1. **建立專案**：用 Astro 建立沛素官網專案，把現有 wireframe 的版面結構轉成 Astro 頁面與元件。
2. **推上 GitHub**：建立一個 repository，把專案程式碼推上去。
3. **連接 Cloudflare Pages**：在 Cloudflare 後台選「Connect to Git」→ 選 repo → 框架選 Astro → 部署。之後每次 GitHub 有更新都會自動重新部署。
4. **綁定網域**：在 Cloudflare Pages 的 Custom Domains 加入你的網域，依指示把 DNS 指向 Cloudflare（若網域 NS 轉到 Cloudflare 管理，設定最順）。SSL 憑證自動簽發。
5. **接上表單**：申請 Web3Forms access key，填入聯絡頁表單。
6. **開啟分析**：在 Cloudflare 後台啟用 Web Analytics，貼上追蹤片段。
7. **架設 CMS 後台**：部署 Sveltia CMS（一個 `/admin` 頁面）＋ 一支 Cloudflare Worker 負責 GitHub OAuth 登入，之後你用 GitHub 帳號登入即可在網頁後台發文。

## 六、日後維護工作流（你自己就能做）

- **發新文章／改產品**：登入 `你的網域/admin` → 在圖形後台打字、上傳圖片、儲存 → 自動上線。
- **改版面或樣式**：交給我，或你熟悉後自行調整 Astro 元件。
- **看數據**：Cloudflare 後台一頁掌握流量；每月可據此提出優化建議。

## 八、實作 Roadmap（建議分階段）

- **階段一（上線 MVP）**：首頁、關於、產品列表、兩支產品內頁、聯絡頁；繁中先行、表單與分析接好、綁網域上線。
- **階段二（內容系統）**：接上 Sveltia CMS，建立專欄文章系統，你可自行發文。
